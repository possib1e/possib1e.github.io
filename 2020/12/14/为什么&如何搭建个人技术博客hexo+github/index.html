为什么&如何搭建个人技术博客hexo github

发表于 2020-12-14 更新于 2020-12-15

为什么建议写个人技术博客，一是记得牢，二是有输出，三是找工作加分。你写博客的过程就是把自己学习、工作遇到的收获、坑点记录下来，因为要对外展示，所以会认真梳理逻辑和细节，相当于再复习一遍，并且这样的文章肯定比你私下的笔记让你更有再次阅读的欲望，想想你之前写过不想再看的代码和笔记。

为什么选用hexo github，因为hexo是用markdown渲染解析文章，方便复制粘贴到其他平台不影响排版，并且可以发布在github上，免费运行，没有广告，页面可以自己随意优化。
https://possib1e.github.io/

本文详细记录自己技术博客hexo github搭建的过程，详细记录中间所有命令和遇到的坑，最终效果如下所示。

2.2 npm安装依赖加速

3. 安装hexo+next

安装hexo+next

npm install -g hexo-cli
git clone https://github.com/theme-next/hexo-theme-next themes/next

修改主题

修改_config.yml
theme: next

初始化博客

初始化
hexo init possib1e
cd possib1e

清除缓存
hexo clean

新建文章
hexo n test_my_site

生成
hexo g

启动服务预览
hexo s

一键部署配置

hexo d

安装如下插件
cnpm install –save hexo-deployer-git
npm install hexo-deployer-git –save

修改_config.yml
deploy:
type: ‘git’
repo: https://github.com/possib1e/possib1e.github.io.git
branch: master

一键上传部署
hexo d

ps:
github上面的仓库名字必须是 你用户名.github.io，具体安装git工具和配置github ssh的过程可以百度。

图片插件

npm install https://github.com/CodeFalling/hexo-asset-image –save

修改主题配置文件_config.yml，生成文章自动生成图像目录

修改Typero
文件->偏好设置，下拉找到 图片插入修改如下，即可粘贴自动存储到目录
./${filename}/

坑点

切换主题报错
ERROR {
err: [Error: EISDIR: illegal operation on a directory, read] {
errno: -4068,
code: ‘EISDIR’,
syscall: ‘read’
}

} Plugin load failed: %s hexo-theme-landscape

解决方法：
修改package.json文件，去掉,”hexo-theme-landscape”: “^0.0.3”这个依赖

中文不生效
修改站点配置文件_config.yml
language: zh-Hans
把theme/next/languages/zh-CN.yml文件改成名字为zh-Hans.yml

中文目录无法跳转bug
https://github.com/theme-next/hexo-theme-next/pull/1540/commits/ec521c927dc10255977324284c1c667f2e220da7

typero换行间距过大
因为typero默认回车是段落的意思，相当于换一行，空一行，所以行间距过大，需要使用shift+enter才是换一行，无法修改，严重影响平时使用的习惯，所以换vscode，微软产品，免费开源，还有很多插件可用。

vscode markdown回车换行不生效
按Ctrl+Shift+P调出搜索栏, 输入setting,找到Open Settings(JSON),在其中添加一行”markdown.preview.breaks”:true.问题解决.
