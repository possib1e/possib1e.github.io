通过学thinkphp开发理解漏洞

发表于 2020-12-18 更新于 2021-01-05

1 为什么学thinkphp开发

最近遇到很多thinkphp的站点，经常存在一些很不应该的漏洞，一直不理解为什么开发会留下这样的漏洞，明明修复很简单的。

个人感觉，这种基于tp开发的站点，一般都是小站点，大部分是个人全栈开发的，完成功能就是一个挑战，更不用说安全性了，开发人员可能需要看手册去复制粘贴修改代码来实现。所以安全与否取决于官方手册的例子。而官方手册给的例子，一般首先都是简单功能的实现，接着才是特殊功能、带有安全性的实现，有的开发者赶时间，一般只看前面的简单例子，发现能满足需求就直接用了。下面举两个例子。

2 通过学开发找到的漏洞

在B站上看了两个thinkphp开发的视频，自己写了个SRC资产后台管理系统，丑是丑了点，但是基本的登陆、添加、上传、数据库的增删改查、前后端之间的数据交互都用到了，发现按照视频的开发套路，会存在如下的漏洞。


2.1 任意文件上传

理论上按照官方开发手册写应该是最安全的，但是我们看下官方的thinkphp5.1完全开发手册，关于上传部分开头是这么写的，完全没有进行任何后缀的验证，有的开发人员就只看到前面，发现复制粘贴修改已经满足需求功能了，就不看后面的资料了。
https://www.kancloud.cn/manual/thinkphp5_1/353946

有文件后缀验证的例子在普通上传文件例子，多文件上传例子后面。

2.2 未授权访问

2.2.1 如何权限验证

1.类的继承
新建一个Base的类，在初始化的initialize进行session或者登陆的判断，然后其他控制器类继承Base类，那么整个类的方法就都有权限验证了。

2.单独写验证
在需要权限验证的类或者方法里面进行session和是否登陆的判断。

3.thinkadmin代码注释
比如thinkadmin这种thinkphp二改的后台项目，基于RBAC标准，进行了封装简化，在相应的操作方法前面写上这种注释即可。
http://doc.thinkadmin.top/thinkadmin-v5/1201770

2.2.2 什么情况存在漏洞

1.普通thinkphp项目，新建cosntroller类，忘记修改继承自Base，默认是继承Controller，并且没有单独的权限验证；
2.基于thinkadmin项目，新建方法时候，忘记添加那段块注释的权限控制。

3.2 项目开发

3.2.1 创建项目

下载thinkphp相关文件，修改权限，注意远程登陆的权限可能是root，但是真正运行php网站的用户是www，所以可能存在权限问题，无法修改文件。

新建后台控制器，负责页面的逻辑，是用户可以直接URL访问到的
php think make:controller admin/Login

创建数据模型，用户数据库操作，相当于封装了数据库表的内容和操作方法
php think make:model admin/AdminModel

创建验证器类，验证提交的数据有效性
php think make:validate admin/LoginValidate

开启调试模式，方便排错

4 thinkphp开发黑客需要知道的知识点

4.1 url背后的路由规则

thinkphp默认的路由规则是pathinfo方式
可以参考视图的渲染
https://www.kancloud.cn/manual/thinkphp5_1/354065

比如我们访问如下链接
http://www.think.com/index.php/admin/index/login
调用的方法是
/www/application/admin/controller/Index.php中的login方法
如果返回的是html页面，则对应的文件是
/www/application/admin/view/index/login.html

比如我们访问如下链接
http://www.think.com/index.php/admin/index/index
调用的方法是
/www/application/admin/controller/Index.php中的index方法
如果返回的是html页面，则对应的文件是
/www/application/admin/view/index/index.html

4.2 html页面可以运行php代码

因为Html是通过php文件进行渲染调用的，所以html中可以运行php代码，我们可以用于留隐藏后门之类的操作,很少人会去检查静态模板是否存在后门。
这部分属于内置标签，参考如下，以下2种写法都可以在html中执行php代码
https://www.kancloud.cn/manual/thinkphp5_1/354089
{php}phpinfo();{/php}
<?php phpinfo();?>
