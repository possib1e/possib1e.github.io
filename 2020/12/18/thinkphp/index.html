1. 为什么学thinkphp开发

因为最近遇到很多thinkphp的站点，经常存在一些感觉很不应该的漏洞，所以在思考为什么呢？

个人猜测，这种tp改的站点，一般都是小站点，一般都是个人全栈开发的，完成就是一个挑战，更不用说安全性了，开发人员可能很多功能点都不会，需要慢慢看手册去复制粘贴修改来实现，所以安全性就在于官方手册的例子，而官方手册给的例子，一般首先都是简单功能的实现，接着才是特殊功能，安全性的实现。下面举2个例子。

1.1 上传漏洞

我们看下官方的thinkphp5.1完全开发手册，关于上传部分开头是这么写的，完全没有进行任何后缀的验证，有的开发人员就只看到前面，复制粘贴修改已经够完成功能了，就不看后面的资料了。
https://www.kancloud.cn/manual/thinkphp5_1/353946

有后缀验证的例子在普通上传文件例子，多文件上传例子后面，

1.2 未授权访问

thinkphp开发对于权限的验证
一种是新建一个Base的类，在里面initialize进行session的判断，然后其他类继承Base类，整个类的方法就都需要有权限才能访问

一种是还是默认继承Controller，但是在自己类里面进行session和是否登陆的判断

一种是比如thinkadmin这种thinkphp二改的后台项目，进行了封装，在相应的操作方法前面写上这种注释即可。
http://doc.thinkadmin.top/thinkadmin-v5/1201770

为什么存在漏洞

1.普通thinkphp项目，新建controller类，忘记修改继承自Base，默认是继承Controller；
2.基于thinkadmin项目，新建方法时候，忘记添加那段块注释的权限控制。

2 正式开发

thinkphp5.1官方完全开发手册
https://www.kancloud.cn/manual/thinkphp5_1/353946

2.1 开发环境

服务器：ubuntu虚拟机
安装宝塔，宝塔新建网站，nginx+php7+mysql
根目录设置为：/www/wwwroot/wwww.think.com/public
创建ftp,mysql,伪静态，去掉跨站，数据库给权限所有人，方便本地操作

开发：物理机win10
安装phpstorm，xshell
phpstorm配置ftp自动上传，位置在tools/deployment

2.2 创建项目

创建项目，下载thinkphp相关文件，修改权限，注意远程登陆的权限可能是root，但是真正运行php网站的用户是www，所以可能存在权限问题，无法修改文件。
composer create-project topthink/think=5.1.39 www.think.com
cd www.think.com
chown -R www.www ./*
chmod -R 750 ./*

新建后台控制器，页面的逻辑，是用户可以直接URL访问到的
php think make:controller admin/Login

创建数据模型，用户数据库操作，相当于封装了数据库表的内容和操作方法
php think make:model admin/AdminModel

创建验证器类，验证提交的数据有效性
php think make:validate admin/LoginValidate

开启调试模式，方便排错
config/app.php中
‘app_debug’ => true,

下载的静态html模板(比如这些资源是admin这个模块用到的)
静载资源js、css、img、lib存放在public/admin/下面
html存放在application/admin/view/login/下面

2.4 thinkphp开发黑客需要知道的知识点

2.4.1 url链接 controller控制器 html静态页面关系:

可以参考视图的渲染
https://www.kancloud.cn/manual/thinkphp5_1/354065

比如我们访问如下链接
http://www.think.com/index.php/admin/index/login
调用的方法是
/www/application/admin/controller/Index.php中的login方法
如果返回的是html页面，则对应的文件是
/www/application/admin/view/index/login.html

比如我们访问如下链接
http://www.think.com/index.php/admin/index/index
调用的方法是
/www/application/admin/controller/Index.php中的index方法
如果返回的是html页面，则对应的文件是
/www/application/admin/view/index/index.html

2.4.2 html页面可以运行php代码

这部分属于内置标签，参考如下
https://www.kancloud.cn/manual/thinkphp5_1/354089

只要在相应的html里面如下2种写法之一，即可在访问时候视图自动渲染执行php代码
{php}phpinfo();{/php}
<?php phpinfo();?>
